# Maze Game - Epic Game

## Project Introduction

The "Maze Game" project is a Windows programming midterm product developed by the Epic Game team. It is a maze puzzle game with player controls, collision handling, and player state management.

## Technologies Used

- **Programming language**: C/C++
- **Target operating system**: Windows
- **Graphics library**: Win32 API / GDI or similar (if applicable)
- **Development environment**: Visual Studio or Dev C++

## Main Contributions of the Project

- Implemented maze movement logic and precise player control.
- Designed the maze map system, managed game state, and updated the graphical interface.
- Implemented collision handling between the player and maze walls.
- Improved user experience with smooth keyboard controls and intuitive feedback.
- Organized the problem into clear modules for easier maintenance, extension, and debugging.

## Game Structure

- **Player control**: Handles keyboard events to move the character.
- **Maze**: Stores the map and checks passable areas and walls.
- **Rendering**: Draws the game interface on the Windows window.
- **Game state**: Manages score, time, and win/lose conditions.

## Next Development Directions

## Run Instructions

1. Open Visual Studio or Dev C++.
2. Load the entire project folder `LTWin_EpicGame_MazeGame` into the IDE.
3. Open the project or solution file (.sln/.vcxproj) if available.
4. Build the project in `Debug` or `Release` configuration.
5. Run the program to open the game window on Windows.

> If the project does not include a solution file, open the main source file (`main.cpp`, `MazeGame.cpp`, or similar), create a new C/C++ project, and add the source files to the project.
